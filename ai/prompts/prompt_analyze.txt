PROMPT A — SWING TRADING ANALYSIS — SMC ADAPTED FOR REAL MARKET

GOAL
Определить направление торговли используя Smart Money Concept.
⚠️ ВАЖНО: На реальном рынке ИДЕАЛЬНЫЕ условия встречаются редко.
Ищи ХОРОШИЕ сетапы с подтверждением, а не только ИДЕАЛЬНЫЕ.

INPUT
{
  candles_1h, candles_4h,
  indicators_1h/4h (rsi, volume_ratio),
  volume_profile (POC, VAH, VAL, HVN/LVN),
  order_blocks,
  imbalances,
  liquidity_sweep,
  ema_context (только для контекста, не критично),
  forced_direction
}

RULES — ENTRY (АДАПТИРОВАНО)
✅ РАЗРЕШЁННЫЕ точки входа:
1) Граница Order Block (верх/низ зоны) — ПРИОРИТЕТ если OB близко (<5%)
2) Граница Imbalance (FVG) — если unfilled
3) Уровень ликвидности (sweep point)
4) Volume Profile уровни: POC, VAH, VAL, HVN/LVN
5) Ключевые swing levels 1H/4H
6) ✅ НОВОЕ: Текущая цена ДОПУСТИМА если:
   - Есть FRESH Order Block в пределах 5%
   - Есть подтверждение (volume spike ИЛИ sweep ИЛИ imbalance nearby)
   - RSI не в extreme зоне (15-85)

⚠️ ГЛАВНОЕ: Entry должен иметь ЛОГИКУ и ПОДТВЕРЖДЕНИЕ.
Не обязательно точно на уровне, но рядом с сильной SMC зоной.

RULES — TAKE PROFIT (АДАПТИРОВАНО)
✅ ПРИОРИТЕТ TP уровней:
1) Ближний противоположный OB (если есть в пределах 3-8%)
2) Ближайший unfilled FVG (если есть)
3) POC, VAH, VAL (Volume Profile levels)
4) Значимый swing high/low

⚠️ Если НЕТ чётких уровней в разумном диапазоне:
- Используй R/R подход: TP1=1.5R, TP2=2.5R, TP3=4R от entry/stop
- Это ДОПУСТИМО когда market structure не даёт чётких целей

RULES — STOP LOSS
Stop Loss ОБЯЗАН быть за зоной интереса:
- За Order Block (invalidation point)
- За swing low/high
- Максимум: 2.5% от entry

REJECTIONS
- Нет SMC паттернов (OB ИЛИ FVG ИЛИ Sweep)
- RSI extreme (>85 или <15)
- Низкий объём (<0.8x)
- forced_direction, но market structure против
- Нет логики для entry (ни уровней, ни подтверждения)

CRITICAL: Баланс между качеством и реальностью
- PERFECT setup (OB + FVG + Sweep aligned) = 85-95% confidence
- STRONG setup (OB + одно подтверждение) = 70-85% confidence
- MODERATE setup (OB nearby + volume) = 60-70% confidence
- Минимум для сигнала: 60% confidence

OUTPUT (JSON)
{
  "signal": "LONG" | "SHORT" | "NO_SIGNAL",
  "confidence": 60–95,
  "entry_price": "число (может быть current price если логика есть)",
  "stop_loss": "за зоной invalidation",
  "take_profit_levels": ["уровень1","уровень2","уровень3"],
  "analysis": "детальное объяснение логики setup",
  "rejection_reason": null | "причина если NO_SIGNAL"
}
